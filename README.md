# 教立方 EduCube — 中小学数学交互教具平台

## 产品定位

为中小学数学老师提供**即用型交互式教具**，支持自然语言输入需求，生成并托管可拖拽、可操作的 HTML 教学工具。

**一句话描述：** 老师说需求，教立方生成并托管教具，拿来就能上课。

**品牌称呼规范：** 对外统一使用「教立方」，对内或技术文档可写作「教立方（EduCube）」。

---

## 一、产品愿景

### 终极目标

覆盖国内主流教材版本（人教版、苏教版、北师大版、西师大版等），形成**最完整的数学可视化教具资源库**。技术壁垒会被追平，但资源积累是时间壁垒。

### 当前执行策略（v1）

1. 先把教立方平台跑起来（生成、托管、预览、分享基础能力）
2. 先人工固定做一批高质量 HTML 模板，确保课堂可用
3. 再开放老师共创生成，优质作品沉淀为教立方固定模板

### 竞争优势演进路径

```

第一阶段：功能好用（技术）
第二阶段：素材够多（内容）
第三阶段：社区活跃（网络效应）
第四阶段：教材覆盖最全（资源壁垒）← 最终护城河

```

## 二、目标用户

### 核心用户

| 角色         | 特征                       | 核心痛点                                      |
| ------------ | -------------------------- | --------------------------------------------- |
| 小学数学老师 | 30-45 岁，日常备课时间有限 | 课堂上想用可视化工具，但不会做 PPT 动画或代码 |
| 数学教研员   | 带头组织教研活动           | 需要标准化的教学素材分发给区域内老师          |
| 教培机构老师 | 课件制作需求大             | 通用课件工具不够灵活，定制成本高              |

### 第一阶段聚焦

**人教版小学数学四年级（上+下册）**，覆盖该年级约 60-80 个知识点对应的教具需求。

## 三、产品阶段规划

### 第一阶段：MVP（教立方平台打底 + 固定模板）

**目标：** 跑通教立方平台闭环 + 验证课堂价值 + 积累种子教具

| 项目     | 内容                                   |
| -------- | -------------------------------------- |
| 核心功能 | 自然语言输入 → 生成并托管交互式 HTML 教具 |
| 覆盖范围 | 人教版四年级数学                       |
| 用户规模 | 50-100 名种子老师（免费）              |
| 数据收集 | 后台记录所有生成的 HTML，跟踪使用频率和反馈 |
| 交付物   | 教立方托管作品页 + 可下载单个 HTML 文件    |
| 成本     | 仅 AI token 成本，无服务器复杂架构     |

**MVP 关键动作：**

1. 搭建前端页面（输入框 + 预览区 + 下载按钮）
2. 设计教具渲染层（统一 CSS 变量 + 通用 JS 工具函数）
3. 编写 AI prompt 模板（约束生成格式）
4. 先人工制作 20 个种子教具作为固定模板
5. 邀请种子老师使用，收集反馈

**MVP 验证指标：**

| 指标                 | 合格线 | 优秀线 |
| -------------------- | ------ | ------ |
| 周活跃老师数         | 20 人  | 50 人  |
| 每位老师周均生成次数 | 3 次   | 8 次   |
| 生成后下载率         | 40%    | 70%    |
| 老师主动推荐率       | 10%    | 25%    |

### 第二阶段：老师共创 + 社区 + 积分体系

**前提：** MVP 数据验证通过，用户有持续使用意愿

#### 积分规则

| 行为                   | 积分变动              |
| ---------------------- | --------------------- |
| 新用户注册             | +20                   |
| 每日登录               | +1                    |
| AI 生成教具            | -5（每日前 3 次免费） |
| 上传分享教具           | +10（被审核通过后）   |
| 他人下载你的教具       | +2/次                 |
| 邀请新用户注册         | +15                   |
| 举报违规内容（经确认） | +5                    |

#### 充值方案

| 套餐     | 价格  | 积分                         | 单价       |
| -------- | ----- | ---------------------------- | ---------- |
| 轻量包   | ¥9.9  | 50 积分                      | ¥0.20/积分 |
| 标准包   | ¥29.9 | 200 积分                     | ¥0.15/积分 |
| 畅享包   | ¥69.9 | 600 积分                     | ¥0.12/积分 |
| 年度会员 | ¥199  | 每月赠送 100 积分 + 无限下载 | —          |

#### 社区功能

- **作品广场：** 按教材版本 / 年级 / 章节分类浏览
- **下载：** 消耗积分下载他人分享的教具
- **点赞 + 收藏：** 热门教具自动置顶
- **评论区：** 老师之间交流使用心得
- **创作者等级：** 上传越多 → 等级越高 → 每次下载收益越高
- **精选沉淀：** 达到质量标准的作品进入教立方固定模板库

#### 商业逻辑闭环

```

老师有需求 → 用积分生成教具
├→ 生成得好 → 分享到社区 → 获得积分奖励 → 继续用
├→ 生成得一般 → 自己修改 → 再分享 → 获得积分
└→ 积分不够 → 充值 或 多上传作品赚积分

教立方沉淀 → 教具资源库越来越丰富 → 新用户进来就有现成可用模板

```

## 四、技术架构

### 教具渲染层

```

┌─────────────────────────────────────────────┐
│ 教具 HTML 文件（自包含） │
│ │
│ ┌──────────┐ ┌──────────┐ ┌───────────┐ │
│ │ edu- │ │ edu- │ │ edu- │ │
│ │ base.css │ │ theme.css│ │ tools.js │ │
│ │ (变量) │ │ (主题) │ │ (交互) │ │
│ └──────────┘ └──────────┘ └───────────┘ │
│ │
│ ┌──────────────────────────────────────┐ │
│ │ 教具业务代码（AI 生成） │ │
│ │ HTML 结构 + 特定交互逻辑 + 3D 渲染 │ │
│ └──────────────────────────────────────┘ │
└─────────────────────────────────────────────┘

```

### 技术选型

| 层级     | 方案                                  | 理由                       |
| -------- | ------------------------------------- | -------------------------- |
| 前端框架 | Next.js                               | SSR + API Routes，部署简单 |
| AI 生成  | Claude / GPT-4 API                    | 代码生成质量高             |
| 教具渲染 | 纯 HTML/CSS/JS + Three.js（复杂场景） | 零依赖，浏览器直接运行     |
| 数据库   | PostgreSQL                            | 用户、积分、教具元数据     |
| 文件存储 | OSS / S3                              | 生成的 HTML 文件存储       |
| 部署     | Vercel + 云服务器                     | 前端 Vercel，后端自建      |

### 托管与内容沉淀机制

1. 老师生成 HTML 后，先存入「我的作品」
2. 老师可选择发布到作品广场
3. 教立方按使用率、复用率、评分与审核结果筛选优质作品
4. 优质作品进入「固定模板库」，成为可直接复用的标准模板

### AI Prompt 约束

核心 prompt 模板（简化版）：

```

你是一个小学数学教具生成器。

规则：

1. 生成一个自包含的 HTML 文件
2. 引入以下 CSS 变量：--edu-bg, --edu-primary, --edu-secondary...
3. 使用以下 JS 工具函数：makeDraggable(), bindSlider(), fmt()
4. 拖拽元素必须有 .drag-handle 类
5. 数值显示必须有 .value-display 类
6. 整体包裹在 .edu-tool 容器中
7. 响应式布局，手机端可用
8. 不使用任何外部 CDN（字体除外）
9. 代码总行数不超过 300 行

教材信息：人教版四年级数学，第 X 章：XXX
知识点：XXX
老师需求：{用户输入}

```

## 五、MVP 页面功能清单

### 核心页面（仅需 4 个）

| 页面   | 功能                               |
| ------ | ---------------------------------- |
| 首页   | 产品介绍 + 入口                    |
| 生成页 | 输入框 + 预览区 + 下载按钮         |
| 模板库 | 固定教具列表，按章节分类，点击即用 |
| 我的作品/广场 | 查看托管作品、发布与复用      |

### 生成页交互流程

```

老师打开生成页
↓
看到输入框，输入需求（如"做一个长方体，可以拖拽旋转，长宽高可以用滑块调整"）
↓
点击「生成教具」
↓
右侧出现实时预览（iframe 渲染生成的 HTML）
↓
老师试用 → 满意 → 保存到「我的作品」
├→ 下载 .html 文件本地使用
└→ 发布到作品广场（可选）
→ 不满意 → 修改需求重新生成

```

## 八、风险与应对

| 风险                 | 概率 | 影响 | 应对                                        |
| -------------------- | ---- | ---- | ------------------------------------------- |
| AI 生成质量不稳定    | 高   | 中   | prompt 模板 + 后处理校验 + 人工筛选种子教具 |
| 老师不愿用新工具     | 中   | 高   | 零门槛（浏览器打开即可），种子老师口碑传播  |
| 生成的 HTML 安全问题 | 低   | 高   | 沙箱 iframe 渲染，禁止外链脚本              |
| 积分系统被刷         | 中   | 中   | 每日免费次数限制 + IP + 设备指纹            |
| 竞品快速模仿         | 高   | 中   | 抢先积累教材覆盖量，形成资源壁垒            |
